# Яккасарой · Yakkasaroy Management System

Веб-приложение управления сетью ресторанов и туйхон «Яккасарой» (Душанбе · Худжанд · Панджакент). Реализует технологию финансового планирования и административную технологию ХМС, с прицелом на масштабирование до 100 заведений к 2030 году.

Тёмная тема с зелёными акцентами в стиле Alif Business, светлая тема, мобильная адаптация. Актуальное ТЗ — [`docs/ТЗ_Yakkasaroy_Management_System_v2.md`](docs/ТЗ_Yakkasaroy_Management_System_v2.md) (уточнения по фондам — `docs/funds-spec.md`).

## Текущий статус

**Приложение работает на Supabase** (PostgreSQL + RLS, Auth) через API-слой `src/lib/api.js`: финансовый контур, сотрудники, оргсхема, статистики, личный кабинет и CRM ходят в реальную БД. На моках остаётся только модуль «Ресторан» — дизайн-референс: по ТЗ v2 §4.11 его заменит интеграция iiko и iframe-модуль yakkasaroy-menu.

## Запуск

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # сборка в dist/
npm test           # unit-тесты (Vitest)
npm run test:e2e   # E2E-смоук (Playwright): экран входа + обход всех вкладок
```

Требуется Node.js 18+.

E2E-смоук гоняется против адреса из `E2E_BASE_URL` (по умолчанию `http://localhost:5173`;
можно указать продакшен/превью Vercel). Авторизованный обход всех модулей и разделов
включается переменными `E2E_EMAIL`/`E2E_PASSWORD` (тестовый аккаунт); без них выполняется
только смоук экрана входа. Локально перед первым запуском: `npx playwright install chromium`.

## Принципы (из ТЗ, менять нельзя)

1. Распоряжаемся только фактически имеющимися средствами (сверка «факт vs расчёт»).
2. Все деньги при поступлении распределяются по фондам по заданным схемам.
3. Расход — только из фонда через одобренную заявку или счёт.
4. ФП проводится еженедельными периодами, период закрывается Директивой.
5. Каждая точка учитывается отдельно, владелец видит консолидированную картину.
