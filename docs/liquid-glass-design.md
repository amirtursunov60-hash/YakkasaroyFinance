# Liquid Glass — дизайн-система Яккасарой

Статус: принято как фундамент дизайна (владелец). Раскатываем эволюционно,
отдельными PR. Этот документ — источник истины по визуальному языку.

## 1. Принципы (неизменные)

1. **Стекло — это «хром», не «данные».** Стекло живёт на навигации, шапках,
   кнопках, гранях карточек, оверлеях. Цифры/строки/таблицы — высокий контраст,
   плоско. Реестр и Директиву стеклом не «топим».
2. **Под стеклом — плоский фон.** Преломление и блики читаются только над
   плоской/контентной подложкой (`pageGrad` плоский, декоративные градиенты убраны).
3. **Выбор — движением.** Активное состояние показывает скользящая пилюля /
   вдавливание, а не перекрашивание. Сегменты, тоглы, табы — единая механика
   «трек + пилюля».
4. **Стекло — per-theme из одного источника.** Всё стекло берёт `theme.glass`
   `{c, light, dark, rl, rd, sat}` (см. `src/theme/theme.js`). Настройка темы
   (в dark глубже, в dim синеватее) меняет всё стекло разом.
5. **Бренд-зелёный — акцент, не материал.** Стекло нейтральное; зелёным маркируем
   смысл («деньги», успех, отдельные активные иконки), а не каждую поверхность.
6. **Мобайл и Safari — первый класс.** Эффект несут блики (`box-shadow`), а не
   `url()`-displacement (в Safari его нет). Бюджет — ограниченное число
   `backdrop-filter` в видимой области.

## 2. Токены и рецепт

- **Источник стекла:** `theme.glass` для light/dark/dim.
- **Рецепт бликов:** `makeCss(C)` в `src/theme/css.js` — `trackShadow` (трек),
  `pillShadow` (пилюля), построены из `L(p)`/`D(p)` (блики `--c-light/--c-dark`
  с множителями `rl/rd`).
- **Уровни стеклянных поверхностей** (один генератор, разная «сила»):
  - `glass-track` — основа сегмента/нав (есть: `.modbar`, `.gseg`).
  - `glass-pill` — бегунок выбора (есть: `.modpill`, `.gseg__pill`).
  - `glass-card` — карточки/панели (мягче, крупный радиус). _(план)_
  - `glass-raised` — модалы/поповеры/шапка (сильнее тень-отрыв). _(план)_
  - `glass-button` + `:active` — капсула + «вдавливание». _(план)_
- **Шкалы:** радиусы 4/8/12/16/20/99; отступы 4/8/12/16/24.

## 3. Инвентарь компонентов → целевое стекло

| Компонент | Сейчас | Цель | Уровень |
|---|---|---|---|
| Табы раздела | ✅ | — | track + pill |
| Свитчер темы | ✅ | — | track + pill |
| Тоглы (Звук, Список/Подробно) | плоские | трек + пилюля | track/pill |
| Фильтр-чипы (Счета, Клиенты) | заливка цветом | сегмент-стекло | track/pill |
| Кнопки (вкл. «Одобрить») | плоский/градиент | капсула + `:active` | button |
| Карточки (фонды, списки) | `panel`+бордер | многослойные блики | card |
| Модалы / поповеры | `panel` | стекло с отрывом | raised |
| Инпуты/поиск | `inputBg` | тонкая грань | card (лёгкий) |
| Шапка / сайдбар | `panel`+blur | хром-стекло | raised |
| **Таблицы, Реестр, Директива (данные)** | плоско | **остаются плоскими** | — |

## 4. Архитектура внедрения

- Рецепт стекла — в `makeCss(C)` (per-theme), классы `.gseg/.gseg__pill`,
  `.modbar/.modpill`; новые уровни добавляются туда же.
- Переиспользуемые UI-компоненты — в `src/components/ui/` на TypeScript
  (`GlassSegment` — трек + скользящая пилюля с замером активной опции).
- Экраны не переписываем ради миграции: **трогаешь компонент → переводишь его
  поверхность на стеклянный уровень** (Boy Scout). Мобильная адаптация и три темы —
  в каждом PR (DoD репозитория).

## 5. Раскатка по PR (каждый — превью → мерж)

1. **G1 · Тоглы и сегменты** — компонент `GlassSegment`, применить к
   Звук/Список-Подробно/фильтрам. _(этот PR — старт)_
2. **G2 · Кнопки** — `glass-button` + `:active` вдавливание; «Одобрить» — стеклянная.
3. **G3 · Карточки** — фонды/списки на `glass-card`.
4. **G4 · Модалы/поповеры/шапка** — `glass-raised`.
5. **G5 · Полировка движения** — единые тайминги/кривые, фокус-кольца, тени-отрывы.

## 6. Гард-рейлы и критерии приёмки

- **Перф-бюджет:** не больше ~3–4 «тяжёлых» `backdrop-filter` в видимой области;
  на мобиле — деградация до блюра.
- **Safari/iOS:** обязательный `-webkit-` фолбэк; эффект не зависит от `url()`-фильтра.
- **Контраст данных:** на стекле — только лейблы/хром; цифры проверяем в трёх темах.
- **Гейты:** `typecheck`/`test`/`lint`/`build` зелёные; стекло — из `theme.glass`,
  никаких случайных `rgba`.
- **Откатываемость:** всё стекло через токены/классы, выключается централизованно.
