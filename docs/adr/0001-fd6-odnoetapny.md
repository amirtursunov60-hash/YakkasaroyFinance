# ADR-0001. Фонд одноэтапный: этап — свойство фонда

- **Дата**: 2026-06-18 · **Статус**: принято (решение заказчика)
- **Решение**: этап распределения задаётся как одно свойство фонда (`funds.stage`);
  «домашний этап» в Директиве = дефолтное правило `distribution_rules`; смена этапа —
  RPC `fp_set_fund_stage`. Прежнее правило «ФД6 пополняется с двух этапов сразу» отменено.
- **Почему**: упрощение модели и UI вкладки «Фонды»; двухэтапный ФД6 путал распределение.
- **Что меняет**: отменяет формулировки ТЗ v2 §1.2/§4.1.3/§10 (там проставлены пометки).
  Первоисточник деталей: `docs/funds-spec.md` §10–12.
