# Журнал решений (ADR)

Каждое значимое решение заказчика или архитектурное решение — отдельный файл
`NNNN-краткое-имя.md`. Это «протез памяти» для ИИ-сессий: незаписанное решение
будет пересмотрено заново, записанное — исполняется без повторных споров.

Правила:
- Новое решение → новый файл (номер по порядку) **и** пометки в документах,
  которые оно отменяет/уточняет (ТЗ, README, funds-spec…).
- Решения не редактируются задним числом: отмена = новый ADR со ссылкой на старый.
- Формат: Дата · Статус (принято/отменено ADR-NNNN) · Решение · Почему · Что меняет.
