# ADR-0003. Инварианты денег — без слепого SERIALIZABLE

- **Дата**: июнь 2026 · **Статус**: принято
- **Решение**: денежные инварианты (запрет минуса, блокировка периода, неизменяемость
  Реестра, защита от двойной оплаты) держит БД: триггеры, констрейнты и точечные
  `SELECT … FOR UPDATE`. Массовый переход на уровень изоляции SERIALIZABLE не делаем.
- **Почему**: SERIALIZABLE даёт retry-конфликты по всему приложению; точечные блокировки
  дешевле и предсказуемее. Клиент никогда не является гарантом инвариантов.
- **Что меняет**: DoD в CLAUDE.md («Деньги — только через Реестр fp_register»);
  примеры реализации — миграции 20260615184000, 20260615185500.
