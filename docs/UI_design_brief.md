# Бриф по UI/дизайну — «Яккасарой» (для дизайн-консилиума)

Документ для внешних дизайнеров: что за продукт, текущий дизайн-язык, технические
рамки, структура, живые экраны и запрос на экспертизу. К брифу приложены 8 скриншотов
(тёмная + светлая темы) — карта файлов в конце.

---

## 1. Что за продукт и кто пользователь
Внутренняя система управления сетью ресторанов и туйхон «Яккасарой» (финансовое
планирование по методологии ХМС, статистики, оргсхема, CRM банкетов). Пользователи
по ролям: **владелец, финдиректор, операционный директор, управляющие точек,
бухгалтеры, сотрудники**.

> **Критично:** управляющие работают в основном **с телефона**. Мобильная версия —
> не «приятный бонус», а главный сценарий использования.

## 2. Текущий дизайн-язык
**«Apple HIG + Liquid Glass»**, тёмная тема по умолчанию + светлая:
- **Стеклянные поверхности** — полупрозрачные панели, градиентный фон страницы,
  светлые грани/блики.
- **Зелёный акцент** (наследие стиля Alif Business), `#3ddc84` на тёмной.
- **Деньги — моноширинным шрифтом** (ровные столбцы цифр), валюта TJS (сомони).
- **Семантические цвета статусов** едины для всех модулей: инфо / предупреждение /
  успех / опасность.
- Иконки — `lucide-react`, скругления по шкале 4/8/12/16/20.

> **Честный нюанс:** была попытка усилить «стекло» (воздушные карточки везде) — её
> **откатили**, т.к. финансовые экраны — это **плотные таблицы данных**, и «воздух»
> мешал читаемости. Есть нерешённое напряжение: **красивое «стекло» vs плотность
> финансовых данных.** Нужен ваш взгляд, как сбалансировать.

## 3. Технические рамки (важно для реализуемости советов)
- **Без Tailwind, без UI-китов** (Material/Ant и т.п.), **без библиотек графиков** —
  все графики свои на SVG.
- Всё оформление — из **единой палитры токенов** + инлайн-стили + глобальный CSS для
  ховеров/анимаций. Никаких хардкод-цветов в компонентах.
- Обе темы (тёмная/светлая) должны работать всегда.
- Язык интерфейса — **русский**, заложен i18n (ru/таджикский).
- **ХМС-термины не переводить и не переименовывать** (фонд, Директива, ЦКП, ЗРС,
  Реестр, статистика, состояния, туйхона, точка, ФП, ФРС, БП).

## 4. Навигация и структура
- Слева — **модули** (Дашборд/Мой кабинет, Финансы, Статистики, Оргсхема, CRM,
  Сотрудники), сверху — **разделы** выбранного модуля (горизонтальные вкладки).
- Навигация пока на состоянии (без адресной строки): **прямых ссылок на конкретный
  экран и кнопки «назад» в браузере пока нет** (известно, решим позже). Интересно
  мнение: критично ли это для UX управленца.

## 5. Экраны (где плотность выше всего)
- **Финансы — ядро, 13 экранов:** Директива (распределение по фондам), Доходы,
  Расходы, Фонды, Контроль средств, Реестр (лента всех операций), Заявки
  (финкомитет), Поставщики/Счета, Клиенты, Зарплата, Отчёты. Много таблиц, сумм,
  процентов, этапов.
- **Статистики ХМС** — графики недельных трендов с цветовыми «состояниями»
  (Власть/Изобилие/Норма/ЧП/Опасность).
- **Дашборд / Мой кабинет, Оргсхема, CRM, Сотрудники** — обзорные/списочные.
- (Модуль «Ресторан» — только дизайн-референс, в продукт не идёт.)

## 6. Комментарий по живым экранам (что видно)

### Сильные стороны
- **Единый узнаваемый почерк.** На каждом экране сверху — градиентная «герой-карточка»
  с ключевыми цифрами. Главное видно сразу.
- **Чёткая денежная иерархия.** Крупные жирные суммы, моноширинные цифры, зелёный =
  приход/рост.
- **Хорошо сделана мобилка.** Формы в одну колонку, крупные тап-зоны, главные действия
  — широкие зелёные кнопки, вкладки скроллятся горизонтально.
- **Статусы и потоки понятны.** Цветные бейджи (В работе/Новая/срочно), задачи как
  поток «Исполнитель → Ответственный», состояния статистик.
- **Доменный язык выдержан** везде.

### Что вынести на обсуждение
1. **Плотность финансовых таблиц на телефоне (Директива).** Названия фондов
   обрезаются («Поставщи…»); в строке чекбокс + код + % + сумма. Тесновато для
   быстрого взгляда. Как балансировать «всё уместить» и «читаемо»?
2. **Светлая тема — контраст.** Заметно бледнее/пастельнее тёмной: бледно-зелёная
   карточка на почти белом, светло-серые подписи. Риск низкого контраста
   (доступность). Тёмная выглядит сильнее.
3. **Зелёный делает слишком много работы:** бренд-акцент, «приход денег», «успех»,
   активная вкладка, состояние «Норма». Возможно, развести брендовый и семантические
   цвета, чтобы значения не размывались.
4. **Герой-карточка занимает много высоты.** Красиво, но на маленьком экране «съедает»
   первый экран, плотные данные уходят ниже сгиба. Делать компактнее/сворачиваемой?
5. **Единообразие чисел.** Где-то «1 342 тыс. TJS», где-то «112 300,00 TJS» — разный
   стиль сокращений. Свести к одному правилу.

### По темам
- **Тёмная** — флагман: насыщенная, премиальная, «финансовая». Система смотрится на
  ней лучше всего.
- **Светлая** — мягкая/пастельная, нужен проход по контрасту и плотности.

## 7. Запрос на экспертизу
1. Общая оценка визуального языка: «стекло» уместно для финансового инструмента или
   слишком «игрушечно»? Куда двигать?
2. Плотность vs читаемость финансовых таблиц на десктопе и **на телефоне**.
3. Консистентность между модулями (карточки, кнопки, таблицы, модалки).
4. Доступность/контраст (тёмная тема, цветовые состояния, цифры).
5. Навигация: нужен ли роутер/deep-linking с точки зрения UX.
6. «Быстрые победы» — что улучшить малой кровью с большим эффектом.
