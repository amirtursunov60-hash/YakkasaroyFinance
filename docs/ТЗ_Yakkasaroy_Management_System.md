## Веб-приложение управления компанией «Яккасарой» (Yakkasaroy Management System)

**Версия:** 1.0
**Дата:** июнь 2026
**Заказчик:** сеть ресторанов и туйхон «Яккасарой», Душанбе
**Основа:** доработка прототипа YakkasaroyFinance.jsx
**Референс функционала:** ManaJet (app.manajet.org) — система управления по технологии Hubbard Management System (ХМС)

---

## 1. Общие сведения

### 1.1 Цель проекта
Создать единое веб-приложение для управления сетью заведений «Яккасарой» (туйхоны и семейные рестораны в Душанбе, Худжанде, Панджакенте), реализующее технологию финансового планирования и административную технологию ХМС, с учётом мультиточечности и масштабирования до 100 заведений к 2030 году.

### 1.2 Текущее состояние
Существует React-прототип YakkasaroyFinance.jsx (тёмная тема, зелёные акценты, стиль Alif Business) с базовыми модулями: Директива/ФРС (распределение по фондам), Доходы, Контроль средств. Прототип — фронтенд без бэкенда, без авторизации, без сохранения данных.

### 1.3 Задача разработки
Превратить прототип в полноценную клиент-серверную систему, взяв за образец функциональную модель ManaJet, но адаптированную под ресторанный бизнес и структуру сети «Яккасарой».

### 1.4 Принципы (из логики ManaJet, сохранить обязательно)
- Распоряжаемся только фактически имеющимися средствами (сверка «факт vs расчёт» в Контроле средств).
- Все деньги при поступлении распределяются по фондам по заданным схемам.
- Расход возможен только из фонда через одобренную заявку или счёт.
- Финансовое планирование проводится периодами (еженедельно), период закрывается Директивой.
- Каждая точка сети учитывается отдельно через метки/филиалы, при этом владелец видит консолидированную картину.

## 2. Роли и права доступа

| Роль | Доступ |
|---|---|
| **Владелец (Учредитель)** | Полный доступ ко всем модулям и всем точкам, включая закрытые фонды (дивиденды, резервы) |
| **Финансовый директор** | Весь блок ФП по всем точкам, закрытые фонды, проведение Директивы, закрытие периодов |
| **Операционный директор / Куратор** | Статистики, задачи, Поток, оргсхема, заявки по своим точкам; финансы — только просмотр |
| **Управляющий точкой** | Свои точки: ввод доходов, подача заявок, статистики своей точки, задачи, сотрудники точки |
| **Бухгалтер** | Контроль средств, оплаты по одобренным заявкам и счетам, ведомости ЗП, без права одобрения |
| **Сотрудник** | Личный кабинет: свои задачи, свои статистики, подача заявок, должностные инструкции, сообщения |

Требования:
- Гибкая настройка прав на уровне модуля, раздела, фонда, вида расхода и конкретной точки.
- Доступ к отдельным фондам и статьям расходов можно открывать конкретным людям (как в ManaJet).
- Приглашение сотрудника по ссылке-приглашению.
- Журнал действий (аудит): кто, что, когда изменил — для всех финансовых операций обязательно.

## 3. Функциональные модули

### 3.1 Финансовое планирование (ядро системы — приоритет №1)

Развитие существующих экранов прототипа. Состав по образцу ManaJet:

**3.1.1 Периоды ФП**
- Финансовая неделя (настраиваемые границы, по умолчанию четверг–среда).
- Статусы периода: открыт / на планировании / закрыт. Операции в закрытом периоде запрещены.

**3.1.2 Доходы**
- Виды дохода как справочник: «Банкеты (туйхона)», «Зал ресторана», «Доставка», «Бар», «Кейтеринг» и т.д.
- Каждый вид дохода помечается меткой точки (Яккасарой Худжанд, Family Душанбе и т.п.).
- К каждому виду дохода привязывается своя схема распределения по фондам: процент и/или фиксированная сумма. Разные схемы для разных видов дохода (разная маржинальность банкета и зала).
- Операции дохода: ручной ввод, импорт, автоматическое создание статистики дохода.
- Мультивалюта: TJS (основная), USD, RUB; конвертация по курсу с историей курсов.

**3.1.3 Фонды**
- Два типа: рабочий (доступен для заявок/счетов) и накопительный (резерв, не показывается в заявках).
- Три этапа распределения:
  1. **От выручки** — фонды закупа продуктов и доставки;
  2. **От маржинального дохода** — дивиденды, кредитные платежи, бонусы;
  3. **От скорректированного дохода** — аренда, ФОТ, реклама, коммуналка, развитие.
- Операции: пополнение, перемещение между фондами, заём фонд→фонд с возвратом, обмен валют между фондами.
- Фонды с ограниченным доступом (только Финдиректор/Владелец): дивиденды, резерв, фонд строительства новых точек.
- Структура папок внутри фондов (группировка по точкам или направлениям).

**3.1.4 Расходы**
- Справочник видов расхода с папками: «Продукты», «ФОТ», «Аренда», «Налоги», «Стройка/ремонт», «Маркетинг» и т.д., с метками точек.
- История всех операций по каждому виду расхода.
- Сводный отчёт по расходам за период / диапазон периодов.
- Доступ к статье расхода — настраиваемый по сотрудникам.

**3.1.5 Заявки на оплату**
- Сотрудник подаёт заявку из личного кабинета: сумма, вид расхода, фонд, точка, обоснование, вложения (фото счёта/накладной).
- Статусы: подана → на планировании → одобрена / отклонена → оплачена.
- Вкладка «Заявки» для бухгалтерии: список одобренных к оплате, отметка факта оплаты, способ оплаты (счёт ДС).
- Комментарии и уведомления заявителю о решении.

**3.1.6 Счета поставщиков**
- Три панели: счета / одобрения / оплаты.
- Приоритет одобрения счетов над заявками при проведении ФП.
- Планируемые (повторяющиеся) счета — аренда, коммуналка, интернет.
- Ассортимент в счёте (закупаемые позиции), возвраты поставщику.
- Импорт/экспорт, печать.

**3.1.7 Счета клиентам**
- Для банкетного направления: счёт клиенту на туй с предоплатой и графиком доплат.
- Три панели: счета / оплаты / отгрузки (проведённые мероприятия).
- Планируемые счета (бронь на будущие даты), частичные оплаты, статус долга клиента.
- Генерация договора/счёта по шаблону (Word/PDF) с подстановкой реквизитов.

**3.1.8 Контроль средств**
- Справочник счетов ДС: кассы точек, банковские счета, карты, эквайринг — с папками по точкам.
- Сверка: фактические остатки (вводит бухгалтер) vs расчётные остатки системы (доходы − оплаты ± перемещения). Расхождение подсвечивается.
- Способы оплаты, траты вне ФП (с отдельным учётом и подсветкой), панель «Подробно» по каждому счёту.

**3.1.9 Директива (проведение ФП)**
- Экран еженедельного финкомитета: поступившие за период средства → распределение по фондам по схемам (с возможностью ручной корректировки) → одобрение счетов поставщиков → рассмотрение заявок (одобрить/отклонить с указанием фонда) → закрытие периода.
- Контроль: нельзя одобрить больше, чем есть в фонде.
- Протокол Директивы — автоформируемый документ по итогам закрытия периода.

**3.1.10 Расчёт безокладной заработной платы (этап 2+)**
- Баллы сотрудникам (должность, стаж, уровень руководства, сложность).
- Стоимость балла = ФОТ периода ÷ сумма баллов. ФОТ не может быть превышен.
- Привязка коэффициента к состоянию статистики сотрудника (справочник состояний ХМС).
- Месячная сводка: аванс, остаток, доплаты, удержания. Экспорт для бухгалтерии.

**3.1.11 Управленческие отчёты**
- ДДС по доходам и расходам; ДДС по доходам и фондам; Прибыль/Убытки (P&L); Поставщики — ДДС и отгрузки; Клиенты — ДДС и отгрузки.
- Все отчёты: фильтр по периодам, точкам (меткам), сравнение периодов, выгрузка в Excel.
- Дополнительно для сети: сравнительная таблица точек (доход, расход, прибыль, ФОТ% на точку).

### 3.2 Статистики (приоритет №1)
- Типы: обычная, групповая (сумма статистик точек), формульная, мульти-статистика, внешняя, автоматическая (из данных ФП — например, доход точки за неделю; из карточек Потока).
- Графики по неделям с трендами, состояния статистик (Власть, Изобилие, Норма, Чрезвычайное положение, Опасность, Несуществование — по ХМС).
- Привязка статистики к посту/сотруднику/точке; ввод данных вручную или авто.
- ИЦО — информационный центр организации: сводная панель главных статистик компании и каждой точки.
- Права доступа к статистикам.

### 3.3 Оргсхема (структура компании)
- 7-отделенческая оргсхема ХМС с постами, отделами, отделениями.
- Назначение сотрудников на посты (один человек — несколько шляп).
- Карточка поста: ЦКП поста, статистики, должностные инструкции, подчинённость.
- Печать структуры. Коммуникационный центр (видно, кому писать по какому вопросу).

### 3.4 Должностные инструкции (шляпы)
- Библиотека документов с папками по отделениям/постам.
- Редактор документов, версии, отметка «изучил» сотрудником (контроль обучения).
- Привязка инструкций к постам оргсхемы.

### 3.5 Задачи
- Постановка задач: ответственный, срок, требование отчёта о выполнении, чек-листы, вложения, комментарии.
- Виды отображения: список, календарь, диаграмма Ганта.
- Повторяющиеся задачи (планировщик/шаблоны), подзадачи.
- Боевое планирование: ежедневный план сотрудника на день, привязанный к задачам и статистикам.

### 3.6 Управление проектами (стратегический блок)
- Вкладки: Долгосрочная цель → Цели → Оргполитика → Стратегический план → Программы (с шагами-задачами и проектами древовидной структуры) → Идеальная картина → ЦКП → Статистики.
- Для «Яккасарой»: программа «100 точек к 2030», программы открытия новых точек как тиражируемые шаблоны (чек-лист запуска заведения).

### 3.7 Поток (канбан бизнес-процессов)
- Доски с этапами и карточками, дополнительные поля на доске и в карточках, метки.
- Авто-действия (при переносе карточки на этап — создать задачу, уведомить и т.д.), шаблоны задач в карточке.
- Типовые доски для сети: «Воронка банкетов» (заявка → дегустация → договор → предоплата → мероприятие → доплата), «Открытие новой точки», «Найм», «Ремонт/стройка».
- Статистики на основе карточек Потока (например, количество заключённых договоров на туй в неделю).

### 3.8 CRM (этап 2)
- Карточки клиентов (физлица — заказчики банкетов; юрлица — корпоративы) и поставщиков.
- Контакты, история взаимодействий (задачи и счета видны в карточке), дни рождения, категории.
- Воронка продаж с настраиваемыми шагами.
- Рассылка (SMS/Telegram/email) — поздравления, напоминания о доплате, маркетинг.

### 3.9 Сотрудники
- Список с архивом, карточка сотрудника, дополнительные поля, группы.
- Назначение прав, постов, баллов ЗП, статистик.

### 3.10 Личный кабинет
- Главная: мои статистики, мои задачи на сегодня, мои заявки и их статусы, сообщения.
- Подача заявок на ФП, боевое планирование, внутренние сообщения.

## 4. Мультиточечность (ключевое отличие от ManaJet)
- Сущность «Точка/Филиал» — первоклассный объект системы (не только метка): город, тип (туйхона/ресторан/кафе), статус (работает/стройка/ремонт), управляющий.
- Любая запись (доход, расход, заявка, счёт, статистика, задача) привязывается к точке.
- Все списки и отчёты фильтруются по точке; владелец переключается между «вся сеть» и конкретной точкой одним кликом.
- Сравнительные дашборды точек.

## 5. Интеграции
1. **iiko** (приоритет): автоматическая подгрузка выручки по точкам в виды дохода (закрытие смены → операция дохода), номенклатура для счетов поставщиков. REST API iiko.
2. **Банки Таджикистана / Alif** — по возможности: выписки для сверки Контроля средств (этап 2–3).
3. **Telegram-бот**: уведомления о статусе заявок, напоминания о задачах, ввод статистик управляющими.
4. **Google Sheets/Excel**: импорт/экспорт во всех ключевых разделах.
5. **Открытое REST API** со Swagger-документацией и API-ключами (как у ManaJet) для будущих интеграций.

## 6. Нефункциональные требования
- **Интерфейс**: тёмная тема с зелёными акцентами (стиль текущего прототипа / Alif Business), язык интерфейса — русский, архитектурно заложить i18n (таджикский в будущем).
- **Адаптивность**: полноценная работа на смартфоне (управляющие работают с телефона) — mobile-first для личного кабинета, заявок, статистик, задач.
- **Производительность**: списки до 100 000 записей, отклик интерфейса < 300 мс на типовых операциях.
- **Безопасность**: HTTPS, хеширование паролей, 2FA для ролей с финансовым доступом, резервное копирование БД ежедневно, журнал аудита финансовых операций.
- **Надёжность**: целевая доступность 99,5%; финансовые операции — транзакционные, двойная запись недопустима.

## 8. Этапы разработки

**Этап 1 — MVP финансового контура (2–3 мес.)**
Авторизация и роли; точки; периоды ФП; доходы и виды дохода со схемами распределения; фонды (3 этапа распределения); расходы; заявки; Контроль средств; Директива; базовые статистики дохода; Telegram-уведомления.
*Критерий приёмки: одна финансовая неделя сети полностью проведена в системе — от ввода доходов до закрытия Директивы, остатки сходятся с фактом.*

**Этап 2 — Управление и люди (2 мес.)**
Счета поставщиков и клиентам (с договорами по шаблону); полный блок статистик с состояниями и ИЦО; задачи + боевое планирование; оргсхема; должностные инструкции; управленческие отчёты.

**Этап 3 — Процессы и рост (2 мес.)**
Поток с авто-действиями; управление проектами (стратегический блок); CRM с воронкой банкетов; интеграция iiko; безокладная ЗП; открытое API.

**Этап 4 — Масштабирование**
Мультиязычность (TJ), банковские интеграции, мобильное приложение (PWA или нативное), аналитика для 20+ точек.

## Приложение А. Глоссарий (ХМС-термины, используемые в системе)
- **ФП** — финансовое планирование; **ФРС** — финансовое распределение средств.
- **Фонд** — запас средств на определённую цель.
- **Директива** — еженедельное распределение средств и рассмотрение заявок финкомитетом.
- **Статистика** — измеримый показатель производства поста/отдела/точки.
- **ЦКП** — ценный конечный продукт; **Шляпа** — должностная папка/инструкции поста.
- **ИЦО** — информационный центр организации; **Боевое планирование** — план сотрудника на день.
- **Туйхона** — банкетный зал; **Точка** — отдельное заведение сети.
