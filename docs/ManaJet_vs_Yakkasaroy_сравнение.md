# ManaJet ↔ Yakkasaroy — живое сравнение и бэклог улучшений

> Дата: 2026-06-21. Источник: живой проход обеих систем под тестовыми аккаунтами
> (ManaJet — `app.manajet.org`, компания Yakkasaroy; Yakkasaroy — `yakkasaroy-finance.vercel.app`).
> Скриншоты экранов собраны в сессии (десктоп + мобайл). Документ переживает сессию —
> на него опираемся при планировании спринтов.

## 0. Краткий вывод

ManaJet и Yakkasaroy строятся на одной методологии (ХМС/ManaJet), и **финансовый
контур у них совпадает по составу вкладок** почти 1:1 (Контроль средств · Доходы ·
Фонды · Расходы · Счета поставщиков · Счета клиентов · Заявки · Директива · Расчёт
зарплаты · Управленческие отчёты). Различие — в **глубине проработки** внутри каждой
вкладки и в **наборе нефинансовых модулей**.

Три тезиса:

1. **ManaJet — зрелый «толстый» десктоп-продукт.** Внутри финансовых вкладок у него
   масса операционных мелочей, которых у нас нет: группировки, настройка колонок,
   статус-фильтры со счётчиками, отгрузки, импорт, 5 видов управленческих отчётов,
   зарплата по баллам. Плюс целые модули (Статистики, Боевое планирование, Задачи с
   Гантом, АдминШкала, CRM, Бизнес-процессы, Сообщения, «Плазма»), которые у нас пока
   моки/заглушки.
2. **Yakkasaroy уже выигрывает в одном важном измерении — мобайл.** ManaJet на телефоне
   — это сжатые плотные десктоп-таблицы с горизонтальным скроллом и мелким шрифтом
   (особенно финансы). Наш финконтур на телефоне (карточки, верхние табы, крупные
   суммы) объективно удобнее. **Это не нужно «догонять» — это нужно защищать.**
3. **Терминология ХМС у ManaJet подаётся ровно теми же словами**, без поясняющего
   слоя. Значит «термины путают людей» — общая болячка, и здесь у нас есть шанс
   обогнать эталон (подсказки/расшифровки рядом с терминами), а не копировать.

Стратегия остаётся прежней (mobile-first закалка ядра), а это сравнение даёт
конкретный, приоритизированный список, **чем дополнить финансовое ядро** и какие
зрелые механики ManaJet перенять в первую очередь.

## 3. Поэкранные заметки UX (что нелогично/неудобно — у нас и у них)

**Yakkasaroy — что улучшить по следам сравнения:**
- В Заявках/Счетах нет быстрых статус-фильтров со счётчиками — пользователь не видит «сколько ждёт рассмотрения» одним взглядом (A1).
- Нет группировки расходов по статьям/отделениям — длинные плоские списки (A2).
- В Доходах сравнение с прошлым периодом только в шапке, не по строкам (A3).
- Зарплата и управленческие отчёты — самые «недокрученные» относительно эталона (B2, B3).

**ManaJet — что у него неудобно (наши потенциальные преимущества):**
- Мобайл — боль: горизонтальные скроллы, мелкий шрифт, тап-цели меньше пальца.
- Термины ХМС без расшифровок — новичок тонет так же, как у нас → **наш шанс**: добавить
  короткие подсказки-тултипы рядом с ФРС/ЗРС/ЦКП/состояниями (дешёвый S-PR, ценность высокая).
- Визуально перегруженный грид: много цветов и рамок, низкая визуальная иерархия.

**Сквозная идея (дёшево, ценно):** микро-подсказки к ХМС-терминам (иконка «?» с
поповером-расшифровкой) — отдельный S-PR, бьёт прямо в «термины путают людей».

## 4. Рекомендованный порядок (один PR за раз)

1. **A1** статус-фильтры со счётчиками (Заявки/Счета) — самый заметный выигрыш на экранах, где живут.
2. **A3** колонка «Предыдущий период» в Доходах.
3. Микро-подсказки ХМС-терминов (см. §3) — дёшево, снимает главную жалобу.
4. **A2** группировка расходов/заявок.
5. **A4** «Долг»/«Доступно» на карточке фонда.
6. Далее по волнам B/C после подтверждения заказчиком.

> Каждый пункт — отдельная ветка `claude/<имя>` → PR в `main`; перед PR зелёные
> `npm ci && typecheck && test && lint && build`; мобайл (isMobile) — часть правки;
> списки — карточками; цвета только из палитры `C`; ХМС-термины не переводить.
