# Карта пробелов ManaJet → Yakkasaroy (по вкладкам)

> Сверка функционала нашего приложения с эталоном ManaJet по его OpenAPI-спеке
> (`api.manajet.org/swagger/docs/v1`, 134 эндпоинта, 179 моделей). Дата: 2026-06-24.
> Метод: модель/эндпоинт ManaJet → наш экран + `lib/api.js` + схема БД.
> Приоритет: **high** = критично для ХМС/денег/ежедневной работы; med; low.
> Объём: S/M/L. Статус «колонка есть, но нет в UI» = быстрая победа.
>
> Закрыто ранее: привязка дохода к клиенту (`counterparty_id`) — PR #147.

Легенда статуса: ❌ нет совсем · 🟡 колонка/таблица есть, нет в UI · 🟠 частично.

> **Журнал закрытого (обновление 25.06.2026, PR #149–#167).** Карта составлена 24.06 и ниже по таблицам статусы местами устарели — фактически закрыто:
> - **Доход §1–3** — лента операций + отмена/правка через сторно (#165).
> - **Заявки §1** частичные оплаты ЗРС (#156), **§2** ввод согласованной суммы (#156), **§3** период оплаты ≠ планирования (#162), **§6** комментарии/переписка (#152).
> - **Счета §1** частичные оплаты поставщиков (#158), **§2** откат оплаты клиента (#160), **§3** вложения у счёта клиента (#160), **§10** архив счёта поставщика из UI (#154).
> - **Фонды §1** два флага подтверждения периода (исполнительный + BAF) (#161).
> - **Реестр §1** фильтр по контрагенту, **§4** по способу оплаты (#149), **§2** журнал аудита (#151).
> - **Статистики §2** проставление `entered_by` (#150), **§3** коридор нормы `min_val`/`max_val` (#153).
> - **Оргсхема §12** пост ↔ точка `location_id` (#155).
> - **Контрагенты §16–18** категории + контакты + экран-справочник (#164).
> - **Задачи §1** задача на пост, **§2** БП ↔ статистика/пост, **§7** `period_id` (#163); **§8** Workflow/Kanban, **§9** настраиваемые колонки, **§11** карточка лида, **§15** drag-and-drop (#167).
>
> Итог по уровням: **Уровень 1 и 2 закрыты целиком**; из **Уровня 3** закрыт Workflow/Kanban (#167), осталось три блока (бухледжер, per-period остатки, рассылки/уведомления). Детальная сводка — в конце файла.
