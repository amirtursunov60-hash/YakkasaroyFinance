## 1. Цель (Goal)

Вкладка «Фонды» — управление запасами средств по целям (фондами): видеть по
каждому фонду сколько денег **реально доступно**, сколько уже **обещано к оплате**,
и каков **баланс займов** между фондами; двигать деньги между фондами и вести
полный журнал движений. Данные — производная Реестра `fp_register`, балансы
руками не пишем.

## 2. Контекст (Context)

- **Роли:** активно работают `owner` и `fin_director` (двигают деньги, создают
  фонды). Остальные — просмотр (по правам RLS / `fund_access`).
- **Устройство:** ПК и телефон (управляющие работают с телефона) — мобильная
  адаптация обязательна.
- **Ограничения предметной области (не нарушаем):**
  - Источник истины движения денег — Реестр `fp_register`; балансы фондов —
    производные леджера, запрет ухода в минус на уровне БД.
  - ФП ведётся недельными периодами (чт–ср); операции в закрытом периоде
    запрещены.
  - Деньги — только через Реестр; вставка операций — серверными функциями
    (парные записи `pair_id` для перемещений/займов).
- **Не нарушаем коды-справочники:** реальные фонды ФД1–ФД9/1 не переименовываем.

## 3. Структура экрана

1. **Шапка (hero)** — период ФП + ключевые показатели.
2. **Операция между фондами** — форма (перемещение / заём с возвратом). Только
   для `owner` / `fin_director`.
3. **Список фондов** — таблица с колонками Остаток · Доступно · Долг + строка «Всего».
4. **Операции с фондами** — журнал всех движений по фондам (без фильтра пока).
5. **Модалки:** выписка по фонду, возврат займа, новый фонд/папка.

## 4. Колонки таблицы фондов и расчёт (СОГЛАСОВАНО)

Оставляем **три** колонки (остальные — Остаток на начало / Приход / Расход —
пока убираем):

| Колонка | Смысл |
|---|---|
| **Остаток** | Только **одобренные, но ещё не оплаченные** суммы из этого фонда (зарезервировано к оплате): ЗРС-заявки из вкладки «Заявки», одобренные, но не оплаченные счета поставщиков, одобренные обязательства и т.п. Ни с чем больше не связан. |
| **Доступно** | Свободные деньги фонда — та сумма, что доступна в Директиве и которой можно реально распоряжаться. |
| **Долг** | Сальдо займов между фондами. **с минусом (−)** = фонду **должны** другие фонды (фонд-кредитор, как ФД6 «Резервы»); **без минуса (+)** = фонд **должен** другим фондам (фонд-должник). |

Внизу таблицы — строка **«Всего»** по колонкам.

### 4.1 Как меняются «Доступно» и «Остаток»

| Событие | Доступно | Остаток |
|---|---|---|
| Распределение дохода в фонд («бросили деньги») | **+** | — |
| Одобрение (заявка ЗРС / счёт / обязательство) | **−** | **+** |
| Оплата ранее одобренного | — | **−** (деньги физически уходят) |
| Перемещение между фондами | **−** у источника, **+** у получателя | — |
| Заём фонд→фонд | **−** у источника, **+** у получателя | — (учитывается в «Долг») |
| Возврат займа | обратно | — |

> Физически в фонде = **Доступно + Остаток** (свободное + обещанное к оплате).

## 6. Операция между фондами / «Добавить» — СОГЛАСОВАНО

Форма «Операция между фондами» и кнопка **«Добавить»** в журнале — это **одно и
то же**. Поле **«Тип операции»** управляет тем, какие поля показаны:

| Тип операции | Поля | Эффект |
|---|---|---|
| **Перемещение** | из фонда → в фонд, сумма | −Доступно у источника, +Доступно у получателя |
| **Заём с возвратом** | из фонда → в фонд, сумма | как перемещение + учёт в «Долг»; возврат через ленту |
| **Приход** | только «в фонд», сумма | ручное оприходование денег в фонд → +Доступно |
| **Возврат** | только «из фонда», сумма | изъятие денег из фонда вообще → −Доступно |

- **Комментарий — обязательный** для всех типов.
- Доступно только `owner` / `fin_director`.

## 7. Журнал «Операции с фондами» — СОГЛАСОВАНО (частично)

- Показываем **все операции по фондам** одной лентой (оплаты заявок, счетов,
  операции с документом, перемещения, займы, возвраты, приход, распределение).
- **Фильтр по датам и поиск — пока НЕ делаем.**
- Поля строки (ориентир ManaJet, адаптируем): дата · источник (фонд) ·
  назначение (статья РД) · тип операции · информация (заявка/компания/способ
  оплаты/счёт) · сумма.

---

## 8. Список фондов: строка и действия — СОГЛАСОВАНО

- Код задаётся при создании фонда (вводится вручную). Системы вложенных кодов
  (ФД1/1) не делаем.
- Слева у фонда — **цветная метка** (цвет задаётся при создании).
- Действия по строке фонда: **Подробно** (выписка) · **Редактировать**
  (название/код/тип и пр.) · **Архивировать** (убрать из списка, история
  сохраняется). Лимит/целевой остаток — **не делаем**.
- **Колонка «Долг»** кликабельна → открывает список займов фонда (кому фонд
  должен / кто должен ему).

## 9. Группы (бывшие «папки») — СОГЛАСОВАНО

- Вместо иконки «папка» — **сворачиваемая секция** (заголовок-раздел в нашем
  стиле, без folder-иконки), фонды внутри сворачиваются/разворачиваются.

## 10. Модалка «Новый / Редактировать фонд» — СОГЛАСОВАНО (частично)

Поля:
- **Код**, **Название**
- **Описание**
- **Этап распределения**: выручка / маржинальный / скорректированный
- **Цвет** — метка фонда (палитра из темы `C`)
- **Точка**
- **Секция** (бывшая папка) — выбрать существующую или создать новую
- ☑ **Приватный фонд** — виден только `owner` и `fin_director`
- ☑ **Запрет вывода** — из фонда нельзя брать деньги (перемещение/возврат/оплата
  наружу запрещены), **приход разрешён**

### Открытые вопросы по модалке
- **Этап как свойство правила, не фонда** (принцип ТЗ §2): этап пополнения —
  свойство правила распределения, а ФД6 пополняется сразу с двух этапов. Значит
  поле «Этап» в модалке должно уметь выбрать **несколько** этапов (или создаёт
  правило распределения). Уточнить: один этап или несколько?
- **Тип «рабочий/накопительный»**: остаётся ли он, или его заменяет галочка
  «Запрет вывода» (накопительный = запрет вывода)?
- **Цвет**: фиксированная палитра-пресеты из темы или произвольный?
