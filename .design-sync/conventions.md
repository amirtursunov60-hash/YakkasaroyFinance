# Яккасарой — дизайн-система (как строить из этих компонентов)

Финансово-административная система сети ресторанов и туйхон (ХМС/ManaJet). Эстетика — Apple HIG / «жидкое стекло»: тёмная тема по умолчанию, полупрозрачные стеклянные поверхности, зелёный бренд-акцент, отдельный зелёный для денег. Интерфейс — русский, валюта — смони (TJS).

## 1. Обязательная обёртка темы

Каждый компонент читает оформление из React-контекста через `useTheme()` → `{ C, st }` (палитра и инлайн-стили). **Без провайдера `st` равен `null` и компонент падает.** Поэтому корень любого экрана оборачивайте в `DSPreviewProvider` (он на `window.YakkasaroyDS`, задаёт тёмную тему, ставит CSS-переменные `--c-*` и глобальные классы):

```jsx
const { DSPreviewProvider, Stat, GlassSegment } = window.YakkasaroyDS;

function Screen() {
  const [period, setPeriod] = React.useState("week");
  return (
    <DSPreviewProvider>
      <GlassSegment
        value={period} onChange={setPeriod}
        options={[{value:"week",label:"Неделя"},{value:"month",label:"Месяц"}]}
      />
      <Stat label="Выручка за неделю" value="148 200,00" unit="смони" />
      <Stat label="Чистая прибыль" value="38 900,00" unit="смони" tone="success" />
    </DSPreviewProvider>
  );
}
```

## 2. Идиома стилей — палитра `C`, не хардкод цветов

Компоненты уже стилизованы. Для своей вёрстки берите цвета ТОЛЬКО из палитры темы через `useTheme()`:

```jsx
const { C, st } = useTheme();
// C.green — бренд-акцент (кнопки/активное)   C.money — денежные суммы (приход/рост)
// C.text / C.sub / C.faint — текст/вторичный/блёклый
// C.panel / C.panel2 / C.line — стеклянные поверхности и границы
// C.danger / C.warning / C.success / C.info — семантические статусы
<div style={{ background: C.panel, color: C.text, border: `1px solid ${C.line}` }}>…</div>
```

Не вписывайте hex-цвета — это ломает тёмную/светлую темы. В Tailwind-разметке используйте мостовые токены: `bg-panel`, `text-money`, `text-brand`, `border-line`, `text-danger/warning/success` (они ссылаются на те же `--c-*`).

**Деньги** форматируйте утилитой `fmt` (точные суммы до дирамов); валюта — смони (TJS). Знак минуса у `Stat` автоматически красится в `C.danger`; для здоровья значения используйте проп `tone` ("success" | "warning" | "danger").

**ХМС-термины не переводить и не переименовывать**: ФП, ФРС, фонд (ФД1–ФД9), Директива, ЦКП, ИЦО, ЗРС, Реестр, статистика, квота, туйхона, точка, неделя ФП (чт–ср).

## 3. Где смотреть правду

- Каждый компонент — каталог `components/<group>/<Name>/` с `<Name>.prompt.md` (как применять + примеры) и `<Name>.d.ts` (контракт пропсов). Читайте их перед использованием.
- Палитра и токены — в бандле (`styles.css` и его `@import`-замыкание). Источник цвета — объект `C` (тёмная/светлая темы).

## 4. Что есть и чего нет

Готовые блоки: `Stat` (сводное число), `GlassSegment` / `ThemeSwitcher` / `MjSwitch` (стеклянные переключатели), `BarsChart` / `StatChart` (свои SVG-графики — без сторонних либ), `ConfirmModal` / `DecideModal` (модалы подтверждения/решения), `ItemCard` / `RequestStatusChips` / `CswRow` (заявки ЗРС), `InfoHint`, `Stub`, `GlobalSearch`, `NotifyBell`, `Login`, экраны ресторана. Прикладные экраны на данных (Доходы/Расходы/Директива/Фонды/Реестр, CRM, Оргсхема, Кадры, Статистики) — это композиции из этих же примитивов.
